API Documentation Preview

Creating an Application

Initiates a new application record. Can be configured as secure or non-secure.

  • URL: https://yourapi.com/create
  • Method: POST
  • URL Params:
    • app_name [required]: The name of the application.
    • secure [optional]: Boolean. Default is false. If true, application requires JWT authentication.
  • Success Response:
    • Code: 201
    • Content: {"app_name": "example", "version": "0.1.0", "secure": true/false}
  • Error Response:
    • Code: 400 Content: {"error": "Missing app_name parameter"}
    • Code: 409 Content: {"error": "Application already exists"}
  • Sample Call: 
    curl -X POST "https://yourapi.com/create?app_name=example&secure=true"

Retrieving the Current Version

Fetches the current version number of the specified application.

  • URL: https://yourapi.com/{app_name}/version
  • Method: GET
  • Success Response:
    • Code: 200
    • Content: {"version": "1.2.3"}
  • Error Response:
    • Code: 404 Content: {"error": "Application not found"}
  • Sample Call: 
    curl https://yourapi.com/example/version

Bumping the Version

Increments the version of an application based on the specified version type (major, minor, patch).

  • URL: https://yourapi.com/{app_name}/bump
  • Method: POST
  • URL Params:
    • type [required]: The type of version increment (major, minor, patch).
  • Success Response:
    • Code: 200
    • Content: {"new_version": "1.3.0"}
  • Error Response:
    • Code: 400 Content: {"error": "Invalid version type specified"}
  • Sample Call: 
    curl -X POST "https://yourapi.com/example/bump?type=minor"

Note: Secure apps require a JWT token to be passed in the header for authorization. Non-secure apps do not require authentication.
