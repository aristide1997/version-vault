Complete API Documentation

Detailed information about all the functionalities of our Version Management API.

Creating an Application

Create a new application entry. Optionally set the application to require secure access.

HTTP Request

POST /create

Parameters

  • app_name (required): The name of the application.
  • secure (optional): If true, requires JWT authentication for version management operations.

Example Request

curl -X POST "https://yourapi.com/create?app_name=newApp&secure=true"

Response

{"app_name": "newApp", "version": "0.1.0", "secure": true}

Retrieving the Current Version

Retrieve the current version of a specified application.

HTTP Request

GET /{app_name}/version

Example Request

curl https://yourapi.com/newApp/version

Response

{"version": "1.0.0"}

Version Management

Manage the application's version through bumping or setting directly.

Bumping the Version

POST /{app_name}/bump

Parameters: type (required): Can be 'major', 'minor', or 'patch'. 

curl -X POST "https://yourapi.com/newApp/bump?type=minor"
{"new_version": "1.1.0"}

Setting the Version Directly

POST /{app_name}/set

Parameters: new_version (required): The new version to set, formatted as 'major.minor.patch'. 

curl -X POST "https://yourapi.com/newApp/set?new_version=2.0.0"
{"new_version": "2.0.0"}

Note: Secure applications require a valid JWT for version bumping and setting operations.
